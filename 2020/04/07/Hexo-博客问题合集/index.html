Hexo 博客问题合集

Hexo 博客问题合集

基于Windows系统，本系列博文主要以实操为主，理论及部分非必要功能请查看Hexo官网文档

学习途径（视频）：B站

概要：本篇博文主要为我在搭建博客所遇到的问题及解决方法

更换博客主题

1.复制主题链接

2.在themes文件夹内，右键选择git bash here，输入命令$ git clone 复制的链接

3.下载完成后在themes文件夹有个hexo-themes-XXX新文件

4.返回上层文件夹，打开_config.yml文件，用编辑器打开，更改theme属性值为hexo-themes-XXX

5.在git bash here输入命令

$hexo clean    //清除缓存

$hexo g        //生成静态网页

$hexo s        //开启本地服务器

[$ npm install hexo-deployer-git –save安装完成后，hexo deploy 无反应解决方案 ]

解决：

‘‘

[hexo deploy 提示FATAL解决方案]

通过hexo搭建博客，正常安装，启动，都没有问题，后来，配置了_config.yml文件，出现了如题错误：

‘‘

最后发现竟然是配置时少写了空格，导致的。这个配置文件真是很奇怪。

每一个配置项都是名称+冒号+空格＋设置参数配置而成。这种就好像shell脚本里面赋值“=”号后面必须没有空格一样，这里必须有空格

[在本地server成功，显示没有问题，但是部署到github上之后，就无法显示主题和图片了]

‘‘
‘‘

解决方案：

其一：

更改一下_config.yml文件，其中的url和root属性。

url: http://www.nideyuming.com/blog/

root: /blog

root 下面应该设置成 /你的项目名 而不是/

然后重新部署一下就成功了。

其二：

问题显著特征：部署（部署成功）到GitHub的博客地址为https：//GitHub账号.github.io/博客名称.github.io

解决办法：修改GitHub账号名称或修改博客名称及本地文件名称

[GitHub pages 提示Your site is having problems building: The tag fancybox on line 77 in landscape/README.md is not a recognized Liquid tag.]

解决方案来自B站码匠笔记：

（前提：配置文件_config.yml的branch为master）

在Cmd输入

$ git status 

$ git checkout -b source         //设置本地分支与远程分支为source

$ git status    

$ git add .

$ git commit -m “update deployer”

$ git status 

$ git push origin source

[$ hexo s 报错]

问题描述：

‘‘

Hexo 报错，ERROR Process failed: _posts/测试文档.md

YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key at line 11, column 1：·····

原因：在“测试文档”第11行错误，在name.md文件的第一行有错误，基本是md语法错误

解决方案：检查标点符号，发现第10行的“ ：”，用了中文输入的“：”，修改为英文输入后，保存部署成功。

【GitHub仓库报错】

现象：

‘‘

起因：

输入命令：

$ git add .

$ git commit -m “init repo” 

$ git push                        //报错

$ git push -f                     //强推

解决方案：

1.（百度，未解决）

首先删除package-lock.json 并同步到git，保证git上面没有lock这个文件，不会出警报；

然后在本地文件夹把package-lock.json 还原回来

找到一个叫做 .gitignore，把package-lock.json贴在这个文件里。（这个文件是专门填写，需

要git忽略掉的文件名字的）

再次同步到git，这个时候就会发现，package-lock.json还是在本地文件夹里，但是不在git上，

所以也就不会报错了。

把package-lock.json删除，重新生成（百度，未解决）

2.查看日志，回溯版本（成功有效）

$ git status 

$ git log                    //查看日志

$ git reset +（commit id）    //删除该commit操作

$ hexo clean                //清除缓存

$ hexo g -d                    //一键部署
