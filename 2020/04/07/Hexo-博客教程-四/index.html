Hexo 博客教程(四)

Hexo 博客教程（四）

基于Windows系统，本系列博客主要以实操为主，理论及部分非必要功能请查看Hexo官网文档

学习途径（视频）：B站

概要：本篇博客我们将进入实操第三部分：部署

部署
1.部署配置，修改参数

修改配置文件_config.yml参数：

deploy：

   Type: git

   repo:  github仓库地址    //地址前包含两个空格

   branch:              //master为写作分支，public为部署分支

2.部署到GitHub（可以同时使用多个deployer）

在GitHub创建博客地址“博客名称.github.io”仓库

安装hexo-deployer-git。

$ npm install hexo-deployer-git --save

‘{% asset_img 6.jpg 图片1 %}‘

若安装不成功（或速度过慢）,可以使用淘宝镜像安装

$ npm install -g cnpm --registry=https://registry.npm.taobao.org        //淘宝镜像

$ cnpm install hexo-deployer-git --save            //安装hexo-deployer-git

$ hexo deploy                                //部署到GitHub

或$ hexo g -d                                //一键生成并部署

打开GitHub博客仓库，显示这个为成功

‘‘

3.数据更改后需生成新文件再部署

$ hexo clean         //清除（本地文件夹中）缓存文件db.json和已生成的静态文件public

$ hexo g            //生成

$ hexo deploy        //部署

博客可以打开，功能正常

‘‘

4.部署到GitHub后，将原文件推送到GitHub

将_config.yml中branch:值改为master //防止建站，GitHub pages报错

$ git init                            //建立一个仓库,本地文件夹生成一个.git隐藏文件

$ git remote add origin 仓库链接    //添加到GitHub仓库

$ git remote -v                        //查看添加

'{%  asset_img 9.jpg 图片4 %}'

$ git status                        //查看当前文件状态

$ git add .                            //将所有文件上传

$ git status                        //查看当前文件状态

$ git commit -m”init repo”            //提交文件

$ git status                        //查看当前文件状态

$ git pull --rebase github仓库链接    //

$ git push --set-upstream origin master --force    //将文件推上github（强行用master写作分支）

将_config.yml中branch:值复原为public 

$ hexo deploy                        //部署到GitHub

打开博客验证，成功打开，没有报错

‘‘

到这里，搭建一个Hexo静态博客，算是完成了，但还会有很多拓展待实现。
