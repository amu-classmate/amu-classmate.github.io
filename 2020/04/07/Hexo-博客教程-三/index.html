Hexo 博客教程(三)

Hexo 博客教程（三）

基于Windows系统，本系列博客主要以实操为主，理论及部分非必要功能请查看Hexo官网文档
学习途径（视频）：B站

概要：本篇博客我们将进入实操第二部分：写第一篇博客

在完成“建站”后，我们可以通过在cmd输入几个简单的命令，就可以看到我们搭建的博客

$ hexo clean

$ hexo generate

$ hexo server

复制那个网址到浏览器打开，我们可以看到

‘‘

在网站里，我们又看到了我们的老朋友“Hello World”，这是建站自动生成的。

接下来本博文将带领我们一步一步的完成我们的第一篇博客

指令

1.创建新文章

$ hexo new “第一篇博客” 创建新文章

2.编写第一篇博客

在notepab++中打开source文件夹的posts文件，我们就可以看到我们新建的第一篇博客.md文件

‘‘

2.生成静态页面（根据_posts文件生成public）

输入命令：

$ hexo generate        //生成静态文件（生成public文件和db.json 文件），简写 hexo g

拓展：

选项    描述

-d    -deploy    文件生成后立即部署网站

-w    -watch    监视文件变动

3.部署服务器

在cmd输入

$ hexo server             //启动服务器，默认http://localhost:4000/，简写hexo s

拓展：

选项    描述

-p    --port    重设端口

-s    --static    只使用静态文件

-l    --log    启动日记记录，使用覆盖记录格式

‘‘

其他：

对博客文章的拓展

打开新建的博客文章时，都会默认包含这些内容

‘‘.

这里用”—”包括起来的内容称为’front-matter’,即前置信息，用于给hexo渲染md文档，指定个别文件变量，除了这三项，Hexo 还提供很多配置项供自由发挥： 

参数        描述    默认值

Layout        布局    

Title        标题    文章的文件名

Data        建立日期    文件建立日期

Updated        更新日期    文件更新日期

Comments    开启文章的评论功能    True

tags        标签（不适用于分页）    

categories    分类（不适用于分页）    

permalink    覆盖文章网址    

keywords    仅用于meta 标签和 Open Graph 的关键词（不推荐使用）

注意：只有文章支持分类和标签，您可以在 Front-matter 中设置。在其他系统中，分类和标签听起来很接近，但是在 Hexo 中两者有着明显的差别：分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次。

添加图片

1.在博客配置文件_config.yml文件中搜索post_asset_folder项，将值改为“turn”

‘‘

这样创建新博文时会自动在文章.md所在目录/source/_posts文件夹内生成一个与文章同名的文件夹。

例如：

新建文章 $ hexo new “图片测试” 则会生成一个图片.md文件与一个名为图片测试的文件夹。

2.将博文用到的图片放到博文同名的文件夹里

3.写博文时插入命令

/*

    '{% asset_img siug [title] %}'        //注意，括号为花括号,符号为英文输入符号

*/

举例说明：

假设我要插入一个为1.jpg的图片

/*
    '{asset_img 1.jpg 图片1 %}'
*/
